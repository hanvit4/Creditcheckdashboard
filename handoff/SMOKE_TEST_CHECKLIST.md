# 릴리즈 스모크 테스트 체크리스트

최종 기준일: 2026-02-25

## 0) 사전 조건
- Supabase Auth Provider 설정 확인 (Google, Kakao, Apple)
- SQL 마이그레이션 적용 확인 (`001`, `002`, `003`, `004`)
- 클라이언트 실행: `npm run dev`

## 1) 로그인 스모크
- [ ] 로그인 화면에서 Kakao/Google/Apple 버튼 노출 확인
- [ ] 각 버튼 클릭 시 OAuth 페이지로 이동 확인
- [ ] 인증 후 앱 복귀 및 세션 유지 확인

## 2) 소셜 계정 연동 스모크
- [ ] 프로필 > 소셜 계정 연동 진입 확인
- [ ] Google 연동 버튼 클릭 → OAuth 이동/복귀 후 목록 반영 확인
- [ ] Kakao 연동 버튼 클릭 → OAuth 이동/복귀 후 목록 반영 확인
- [ ] Apple 연동 버튼 클릭 → OAuth 이동/복귀 후 목록 반영 확인
- [ ] 이미 연동된 provider는 추가 버튼이 숨김 처리되는지 확인
- [ ] 마지막 1개 계정만 남았을 때 해제 불가 안내 확인

## 3) 소셜 계정 해제 스모크
- [ ] 2개 이상 연동 상태에서 1개 해제 성공 확인
- [ ] 해제 후 목록에서 즉시 반영되는지 확인
- [ ] 해제 실패 시 에러 토스트 노출 확인

## 4) 교회 등록 스모크
- [ ] 교회 목록 검색/도시 필터 동작 확인
- [ ] 목록에서 교회 선택 후 등록 성공 확인
- [ ] 코드 등록(6자리) 성공 확인
- [ ] 직접 등록(교회명/주소 필수) 성공 확인
- [ ] 교회 소속 삭제 성공 확인
- [ ] 최대 2개 제한 UX 노출 확인

## 5) 기본 데이터/화면 스모크
- [ ] 홈/성경/프로필 탭 전환 정상 동작
- [ ] 프로필 정보(닉네임/교회명) 표시 정상
- [ ] 성경 본문 조회/검색 API 정상 응답

## 6) API 헬스 체크
- [ ] `GET /functions/v1/server/server/health` 200 응답
- [ ] 인증 필요 API 호출 시 토큰 누락 401 확인

## 7) 회귀 체크
- [ ] `npm run build` 성공
- [ ] 소셜 연동 관련 중복 엔드포인트 없음 (`/user/providers/disconnect-all` 1회 선언)

## 판정 기준
- P0(로그인/연동/교회등록) 항목 전부 통과 시 배포 가능
- 실패 항목은 재현 경로/계정/provider/시간을 함께 기록
